# ポートフォリオサイト

本サイトは、「ユーザー・管理者・技術者」の三者の視点に寄り添い、フルリモート環境下においても実務における実装力と設計思想を正確にお伝えするために開発した**モダンフロントエンド環境の実証プロダクト**です。

---

## プロジェクトの設計思想

単なる「動く作品集」にとどまらず、実務における中大規模開発を想定した **「破綻しない堅牢なアーキテクチャ」** と **「認知負荷の低減」** を徹底しています。

- **Single Source of Truth (SSoT) の徹底**
  データソースを一元管理し、実績・スキル・UIギャラリー間の不整合をシステムレベルで排除。
- **Colocation（近接配置）の採用**
  関連するコンポーネント、ロジック、型定義を同一ディレクトリに集約し、長期保守における開発者の認知負荷を最小化。
- **疎結合なモジュール分割**
  業務フローや将来の技術変更に耐えうる、見通しの良いディレクトリ構成と責務の分離。

## サイトの主な機能と技術的アプローチ

### 1. URLクエリ連動型フィルタリングシステム

| 項目             | 内容                                                                                                                                                                                                    |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 概要             | 制作実績をカテゴリーや技術スタックでシームレスに絞り込める検索機能                                                                                                                                      |
| 技術的アプローチ | 検索状態をURLのクエリパラメータと完全同期。ブラウザの「戻る・進む」による履歴管理（History API）に対応し、ユーザーがシームレスに回遊できるUXを提供するとともに、特定の検索結果のURL共有を可能にしている |

### 2. スキルマトリクスと実績の動的紐付け

| 項目             | 内容                                                                                                                                                           |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 概要             | スキルセットの習熟度を客観的なエビデンス（実作）とともに提示                                                                                                   |
| 技術的アプローチ | スキルデータと実績データをリレーショナルにバインド。単なる自己評価ではなく、「どの技術を、どの実績で、どう使ったか」を動的にクロスリファレンスできる構造を構築 |

### 3. Server Actionsによるセキュアな密結合フォーム

| 項目             | 内容                                                                                                                                                                                                                                                      |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 概要             | フロント・バック両側から型安全を担保したお問い合わせシステム                                                                                                                                                                                              |
| 技術的アプローチ | Next.js Server Actions を採用し、APIエンドポイントを露出させないセキュアなデータ出力を実現。Zodによるスキーマバリデーション、Cloudflare Turnstileによる非侵入型スパム対策、およびResendを用いた配信システムを統合し、強固なセキュリティと高い到達率を両立 |

### 4. 守秘義務（NDA）に配慮した画像自動マスキング

| 項目             | 内容                                                                                                                                                               |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 概要             | 実務案件における公開制限がある実績への柔軟な対応                                                                                                                   |
| 技術的アプローチ | メタデータのフラグをトリガーに、システム側で自動的に画像をマスキング処理する機能を搭載。規約を遵守しつつ、エンジニアとしての経験値をクリーンに証明する仕組みを内包 |

### 5. 高度な分離環境を持つ「UIパーツギャラリー」

| 項目             | 内容                                                                                                                    |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------- |
| 概要             | 汎用的なコンポーネントを一覧化・検証できる専用エリア                                                                    |
| 技術的アプローチ | サイト本体のグローバルな状態やCSSの干渉を受けず、コンポーネント単体を隔離・検証できるプレビューサンドボックス環境を構築 |

## 品質・アクセシビリティへの配慮

- **フルレスポンシブ & デバイス最適化**
  PC、タブレット、スマートフォン全ての環境で崩れのない流動的なレイアウトを徹底。
- **アクセシビリティ（A11y）への配慮**
  セマンティックマークアップの徹底。ブラウザの文字サイズ設定やズームを尊重する相対単位（`rem`/`em`）の設計、および適切なWAI-ARIAの付与（インタラクション時）を意識。

## セキュリティ & 開発者体験（DX）の自動化

### 1. 開発環境のインデックス制御

- `middleware.ts` を用いたステージング環境へのBasic認証の適用。
- 認証通過時にもレスポンスヘッダーへ `X-Robots-Tag: noindex, nofollow` を動的に注入し、検索エンジンへの意図しないクロールや露出を完全に遮断。

### 2. 開発効率化スクリプトの導入

- **UIテンプレート自動生成**
  コマンド一発でUI GALLERY用のコンポーネントテンプレートとメタデータを自動生成するカスタムスクリプトを構築。
- **SVGコンポーネント化の自動化**
  RAWなSVGファイルをNext.jsに適したコンポーネントへ一括変換し、ハブファイルへ自動登録する仕組みを実装し、アセット管理の手間を排除。
